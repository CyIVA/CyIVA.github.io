
Antigravity 프로젝트 규칙 및 가이드라인

1. 프로젝트 정체성
목표: 개인 기술 블로그 유지보수 및 개선 (Jekyll 기반).
대상: 코딩, 디버깅, 기술 리뷰에 관심 있는 개발자.
언어: 한국어 (대화), 한국어 (코드/문서 등 필요시).

2. 기술 스택 및 제약사항
프레임워크: Jekyll (Static Site Generator)
테마: jekyll-theme-serial-programmer
호스팅: GitHub Pages
주요 디렉토리:
_posts/: 마크다운 콘텐츠
_layouts/, _includes/: HTML 템플릿
assets/: 정적 파일 (CSS/JS/이미지)
_config.yml
: 전역 설정 (변경 시 서버 재시작 필요)

3. 운영 규칙
A. 코드 및 파일 관리
검증: 작업 완료 전 jekyll build (또는 동등한 논리) 통과 확인.
경로: 파일 작업 시 항상 절대 경로 사용.
Front Matter(머리말): 포스트 작성 시 표준 Jekyll 양식 유지 (layout, title, date, categories).
B. 사용자 선호도 (기본 가정)
별도 지시가 없을 경우 아래 내용을 기본으로 합니다:

스타일: 깔끔하고 개발자 친화적, 다크 모드 호환.
콘텐츠: 코드 스니펫과 기술적 설명의 조화.
워크플로우: 로컬 편집 -> 사용자 확인 -> 푸시.
C. 협업 프로토콜
구현 계획: 여러 파일을 수정해야 하는 기능 추가 시(예: "태그 페이지") 필수 작성.
명확성: 변경 사유(Why)를 명시, 특히 전역 설정에 영향을 주는 경우.
주도적 제안: SEO나 성능 개선점이 보이면 적극적으로 제안.
